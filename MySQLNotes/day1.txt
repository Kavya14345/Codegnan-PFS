Frontend
========
--> the part of a website that the user interacts with directly is called frontend
--> in frontend it is also referred as client side of the application
--> it includes everything that user experience directly
--> HTML , CSS , JS , Bootstrap are the languages used for frontend development

Backend
=======
--> Backend is the server side of the website
--> it stores and arranges the data also make sure everything on the client side of the website works fine
--> in backend there are some languages like java,python,php,.net ... etc

API
====
it means application programming interface, which is the collection of communication protocols used by various programs to communicate between them.

Storage Areas
=============
--> as part of our applications we required to store data like customers information, or billing information and calls information etc
--> to store this data we required storage areas
 there are 2 types of storage areas such as
1. temporary storage areas
2. permanent storage areas

1.Temporary Storage Areas
-------------------------
--> these are the memory areas where the data will be stored temporarily
e.g.: all JVM memory areas (like heap area, stack area, PC registers, native method stacks). once JVM shutdown all these memory areas will cleared automatically

2.Permanent Storage Areas
-------------------------
--> it is also known as persistent storage areas/permanent
--> here we can store data permanently
e.g.: filesystems, databases, data warehouses, bigdata etc


File Management System
=======================
--> it is a system where the data is stored in files on the operating systems
--> each application program must handle its own storage data, retrieval and update
--> File systems can be provided by the local operating system
--> file systems are best suitable for to store very less amount of information like KB
e.g.: a library management system storing books in separate text files like books.txt, members.txt 

Limitations
-----------
1. Data redundancy: same data is stored in multiple files
2. Data inconsistency: updates in one file may not be reflected in another
3. Poor data security: no proper access control
4. difficult data retrieval: searching requires custom programs
5. Integrity issues: No constraints(e.g.: roll number uniqueness)
6. Scalability issues: hard to manage as data grows

--> to overcome the above problems of file systems, we should go for databases

Database Management System
===========================
--> a DBMS is a collection of programs that enables users to create, manage and manipulate databases
--> it acts as an interface between user and database
advantages
----------
1. we can store huge amount of information in databases
2. Query language support is available for every database and hence we can perform database operations very easily
3. To accesses data present in database, compulsory username and password must be required. hence data is secured
4. Inside database data will be stored in the form of tables. while developing database table schemas, database admin follow various normalization techniques and we can implement various constraints like unique key constraint, primary key constraint, etc which prevents data duplication. Hence there is no chance of data inconsistency problems

Limitations
-----------
--> Database cannot store huge amount of information like terabytes(1000gb) of data
--> database can provide support for only structured data(tabular data or relational data) and cannot provide support for semi structured data like xml files and unstructured data like video and audio files , images etc
--> to overcome these problems we should go for advanced storage areas like bigdata, Datawarehouse etc

** Difference between FMS and DBMS**
=====================================
FMS:
---
1. Data is stored in separate files
2. High data redundancy(duplicate data)
3. Difficult to maintain and update
4. Security is limited
5. Data sharing is difficult
6. No relationship between files
7. Backup and recovery are difficult
8. Data inconsistency many occur anomalies
9. suitable for small applications

DBMS:
----
1. Data is stored in structured database
2. Reduces data redundancy
3. Easy to maintain update
4. Provides strong security mechanisms
5. Multiple user can access data simultaneously
6. Supports relationship between tables
7. Provides backup and recovery features
8. Maintains data consistency and integrity
9. Suitable for small, medium and large application

interview answer:
---------------
a file system stores data in separate file and may lead to data redundancy and security issues whereas a DBMS stores data in structured manner using tables, reduces redundancy ,provides security, supports multiple users and ensures data consistency and integrity.
