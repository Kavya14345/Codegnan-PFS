Anchor tag: it is used for routing or navigation to other pages or websites
==========
1.external routing: it is used to navigate to other websites or pages outside the current website or project
absolute url: complete url with protocol and domain name(external links or c:/users/username/desktop/file.html)
syntax:
------
<a href="c:/users/username/desktop/file.html">local file</a>

href attribute: It is hyperlink reference attribute which is used to specify the url of the page the link goes to,by default it will open in the same tab or window

target attribute: it is used to specify where to open the linked document.
syntax: <a href="url" target="_self">link text</a>
        <a href="url" target="_blank">link text</a>

2.internal routing: it is used to navigate to other pages within the same website or project
relative url: relative to the current page location
syntax: <a href="day1.html">day1</a>

Display Behaviour
====================

1. Block-level elements:
-----------------------
this elements start on a new line and take up the full width available. 
Examples include <div>, <p>, <h1> to <h6>, <ul>, <ol>, <form> and <li>.

2. Inline-level elements:
-----------------------
this elements do not start on a new line and only take up as much width as necessary.
Examples include <span>, <a>, <strong>,<img> , input and <em>.

------------------------
div and span tags:
-----------------------
div and span tags are used to group elements together.
1. division elements are used to group the sections and apply CSS styles to many elements at once.(sections)
   <div style="background-color: lightblue;color: black; padding: 10px;">
      <h2>This is a heading inside a div</h2>
      <p>This is a paragraph inside a div.</p>
   </div>
2. span elements are used to group text and apply CSS styles to many elements at once.(texts)
   <span style="background-color: yellow; padding: 5px;">This is a span element.</span>

Semantic HTML: all are closing tags are required for these elements.
-------------------------------
# Semantic HTML elements are those that clearly describe their meaning in a human- and machine-readable way.
<header> - Represents introductory content, typically a group of introductory or navigational aids.
<footer> - Represents a footer for its nearest sectioning content or sectioning root element.
<article> - Represents a self-contained composition in a document, page, application, or site.
<section> - Represents a standalone section of functionality contained within an HTML document, typically with a heading.
<nav> - Represents a section of a page that links to other pages or to parts within the page: a section with navigation links.
<aside> - Represents a portion of a document whose content is only indirectly related to the document's main content.
<main> - Represents the dominant content of the <body> of a document.
<figure> - Represents self-contained content, frequently with a caption, and is typically referenced as a single unit.

Forms
---------
-> To collect the data from the user and allowing users to submit the data to the servers
eg: login form, registration form, contact us form, etc.
-> the <form> tag is used to create a form in HTML. It can contain various input elements like text fields, checkboxes, radio buttons, submit buttons, etc.
-> form fields are defined using the <input> tag, which can have different types like text, password, email, number, etc.

Attributes:
1. Type: Specifies the type of input field (text, password, email, etc.)
2. Placeholder: Provides a hint or example of what the user should enter in the input field
3. for and id: The for attribute in the <label> tag is used to associate the label with a specific input field. 
               The id attribute in the <input> tag is used to uniquely identify the input field. When the user clicks on the label, it will focus on the associated input field.
4. Action: Specifies the URL where the form data should be sent when the form is submitted. If not specified, the form will submit to the same page.
5. name: Specifies the name of the input field, which is used to identify the data when it is submitted to the server.
6. value: Specifies the initial value of the input field. It can be used to pre-fill the field with a default value.
7. required: Specifies that the input field must be filled out before submitting the form. If the field is empty, the form will not be submitted and an error message will be displayed.
