==============================================Constraints============================================================--> Constraints are rules applied to a table columns in MySQL to ensure that only valid and accurate data is stored in the database.Why do we use Constraints?--------------------------1.Maintain Data Integrity2.To prevent invalid data entry3.Avoid Duplicate Records4.Ensures relations between tables5.Make the database reliable and constraintTypes of Constraints:======================1. not null            : it doesn't allow null values but can allow duplicate values.mysql> use pfs3;Database changedmysql> create table students(    -> Sno int not null,    -> name varchar(30),    -> marks int    -> );Query OK, 0 rows affected (0.02 sec)mysql> Actions define what happens to child table data when parent table data is updated or deleted.they are used with foreign key constraints;On Delete Cascade--> when a parent table row is deleted , all matching child rows are automatically deleted.2. On Update Cascade--> when a parent key is updated , all matching child foreign keys are updated automatically.mysql> create table departments(    -> dept_id int primary key,    -> dept_name varchar(30)    -> );Query OK, 0 rows affected (0.02 sec)mysql> create table employees(    -> emp_id int primary key,    -> name varchar(100),    -> dept_id int,    -> foreign key(dept_id)
